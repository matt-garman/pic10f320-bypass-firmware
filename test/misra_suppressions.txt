# cppcheck per-file suppressions for the MISRA-C:2012 analysis (make analyze-misra).

# This file holds DOCUMENTED, PROJECT-SPECIFIC deviations: MISRA rules we
# knowingly do not satisfy in our own source. Every entry must map to a numbered
# deviation record ("D-n") in MISRA_COMPLIANCE.md, which carries the rationale.
# A suppression here without a matching D-record there is a defect -- the whole
# point of MISRA compliance is that every deviation is justified.

# Suppressions are PER FILE (errorId:filename), not project-wide, so a new
# instance of a waived rule in a NEW file still surfaces and fails the analysis.

# Out of scope (handled on the cppcheck command line in the Makefile, not here):
# violations inside the XC8 / DFP system headers -- the toolchain library is
# outside this project's compliance boundary.

# NOTE: a line that is exactly "#" is parsed by cppcheck as a malformed
# suppression ("Failed to add suppression. No id."), so the separator lines in
# this file are left genuinely empty rather than a bare "#".
